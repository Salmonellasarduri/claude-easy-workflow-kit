# /restart — 中断後の自動復旧

## 目的

セッションが中断された後、どこで止まったかを検出して自動的に再開する。

---

## 実行手順

### 1. 現在の状態を確認する

```bash
git status
git log --oneline -5
```

タスクファイル（`workflow.yaml` の `paths.tasks`）の最新状態も確認する。

### 2. ケース分岐

#### ケース A: コミットされていない変更がある

```
git status に変更ファイルが表示される
  ↓
変更内容を確認する (git diff)
  ↓
変更が意図したものか判断する
  ↓
意図した変更 → /review → /save を完了させる
意図しない変更 → ユーザーに確認する
```

#### ケース B: 未完了のタスクがある

```
タスクファイルに [ ] のタスクが残っている
かつ git status がクリーン
  ↓
どのタスクまで完了しているかを特定する
  ↓
次の未完了タスクから /implement を再開する
```

#### ケース C: すべて完了している

```
タスクファイルの現フェーズタスクがすべて [x]
git status がクリーン
  ↓
ユーザーに完了を報告して次のアクションを提案する
/strategy を実行して次のフェーズを提示する
```

---

## 出力フォーマット

```
## 復旧状態確認

現在の状態: [A / B / C]

### 検出した状況
- [具体的な状態の説明]

### 実行するアクション
- [次に取る具体的なステップ]

再開してよいですか？
```
