# /implement — 実装・デバッグループ

## 目的

承認済みのプランを実行し、動作確認が取れるまでデバッグループを回す。

## 前提条件

- `/plan` が完了しユーザーに承認されていること
- タスクファイル（`workflow.yaml` の `paths.tasks`）に以下が揃っていること:
  - 変更ファイル一覧
  - 実装ステップ（順序付き）
  - 受け入れ基準

---

## 実行手順

1. タスクファイルの現在タスクから以下を確認する:
   - 変更ファイル（実装対象を限定する）
   - 受け入れ基準（完了判定に使う）

2. プロジェクトのコーディング規約に従って実装する

3. 実装後に **`/debug`** を実行する:
   - テストデータ投入 → 検証 → クリーンアップの全手順は `/debug` に従う
   - 完了条件（受け入れ基準証明・テストデータ除去）を満たすまでループする

4. `/debug` 完了後 → `/review` を呼ぶ

---

## 完了条件

- [ ] 受け入れ基準がすべて満たされている（`/debug` で証明済み）
- [ ] テストデータがすべて除去されている
- [ ] エラーログに警告・エラーが出ていない
- [ ] `/review` を通過した
