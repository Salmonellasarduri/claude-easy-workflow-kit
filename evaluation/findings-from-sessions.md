# セッションログから見えた CEWK ワークフローの実践的価値

> 7,666セッション（Artificial Personality プロジェクト）から、strategy_deep 使用220セッションを中心に抽出。

---

## 発見1: マルチエージェント批評が「見えない前提」を叩き出す

### 実例: self.md 硬直問題

**ユーザーの依頼:**
> 「カーリー、バックのような初期の読書の具体的な経験だけ入っていて、アンバランスな想起をする」

**Claude の初期戦略:** エピソードに鮮度を追跡、古いものを退場候補にする

**Codex の Critical:**
> 「鮮度と重要度の混同 -- 14日未強化=退場は人格の中核を壊す」

**Gemini の Critical（Round 2）:**
> 「聖人化（The Wisdom Trap）-- 具体から抽象を繰り返すと予測可能で面白みのない人格になる。未消化の傷跡こそが魅力」

**結果:** 「鮮度ベースの退場」が3ラウンドで「三状態代謝モデル（Bone/Scar/Living）」に進化。Codex なしでは二層構造止まり、Gemini なしでは「傷跡」概念が生まれなかった。

## 発見2: Critical の「質が変化する」-- 問題の本当の形が見えてくる

| ラウンド | Critical | 本質 |
|---------|----------|------|
| R1 | エピソード同一性管理が不可能 | 追跡の粒度問題 |
| R2 | 主張 provenance 不足 | R1を解決したが新しい問題が出現 |
| R3 | Bone層に先住者優位が移動するだけ | R2の解決策がR1の問題を移しただけ |

**R3での根本的再認識:**
> 「問題の本質は抽象化が足りないではなく先住者優位（incumbency advantage）だった。最初に入った経験がシステムの未成熟さゆえに不動の地位を得ている」

1ラウンドでは「正しい方向の不十分な解」で終わる。複数ラウンドの批評で問題の本当の形が見えてくる。

---

## 発見3: 判断の根拠が可視化される -- 「選びやすさ」の正体

### 実例: レイテンシ改善の選択肢提示

| Tier | 目標 | 実測推定 | 主要ボトルネック |
|------|------|----------|------------------|
| T1 | <1s | 700-900ms | Haiku API roundtrip |
| T2 | 2-5s | 2.5-3.5s | Sonnet tool_use 1回 |
| T3 | 5-20s | 5-15s | Web grounding + Opus CLI |
| T4 | 20s+ | 10-25s | think -> _think_deep -> expand_nucleus の3段直列 |

「A: 速い案 / B: 安全な案」のような抽象二択ではなく、数値付きの根拠表として提示される。判断の数が少ないのではなく、判断の根拠が見える。

---

## 発見4: 仕様遵守 + 品質検証の一貫性

### カードゲーム比較での観察

| 環境 | 仕様遵守 | 品質検証 |
|------|---------|---------|
| 素 Claude Code | ○ | なし |
| superpowers | × (brainstorm で仕様変質) | なし |
| CEWK | ○ | /debug 8項目 + /review 9項目 |

「仕様が変わったことに気づけない」のは文系ユーザーにとって最大のリスク。CEWK はそのリスクを構造的に排除する。

---

## CEWK の本当の強み

1. **マルチエージェント批評で見えない前提を炙り出す** -- Claude 単体では到達できない設計品質
2. **複数ラウンドで問題の本当の形を発見する** -- 1ショットでは見えない構造的欠陥
3. **判断の根拠を可視化する** -- 選びやすさの正体は判断数の少なさではなく根拠の明確さ
4. **仕様遵守 + 品質検証の一貫性** -- 創造性よりも「頼んだものが頼んだ通りにできる」安心感
