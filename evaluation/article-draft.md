# Claude Code ワークフローキット比較：文系でも「選ぶだけ」で開発できるのか

> 素の Claude Code / superpowers / claude-easy-workflow-kit で同じタスクを投げて、何が起きるか観察した。

---

## この記事の前提

筆者はプログラミング経験のない人間が Claude Code で開発を回す方法に興味がある。

世の中には Claude Code を強化するワークフローキットがいくつかある。最大手は **superpowers**（GitHub 130k★）。14個のスキルと TDD 哲学で「コーディングエージェントに超能力を与える」ツールだ。

一方、筆者が使っている **claude-easy-workflow-kit**（以下 CEWK）は、`/strategy` と打つだけで計画→実装→テスト→レビュー→保存まで自動で進む、7コマンドの連鎖型ワークフロー。判断ポイントは「A/B/Cを選ぶ」と「はい/いいえ」だけ。

**疑問：文系ユーザーが空のフォルダから始めて、迷わず使えるのはどちらか？**

この問いに答えるために、同じタスクを3つの環境で実行して比較した。

---

## 実験の設計

### タスク

> 対話型のカードゲームを作ってほしい。プレイヤーと CPU が交互にカードを出して、数字が大きい方が勝ち。5ラウンドやって勝った数が多い方の勝ち。手札はランダムに5枚。プレイヤーがどのカードを出すか選べる。結果の履歴を画面に表示して、見た目もかっこよく。

### 3つの環境

| 環境 | 導入方法 | 開始コマンド |
|------|---------|-------------|
| **素の Claude Code** | なし | プロンプトをそのまま入力 |
| **superpowers** | スキルとコマンドを手動配置 | `/brainstorm` + プロンプト |
| **CEWK** | scaffold.ps1 で配置 | `/strategy` + プロンプト |

条件：同一マシン、同一モデル（claude-opus-4-6）、導入済みの状態でスタート。

---

## 結果

### 素の Claude Code：1分で完成。選択権はゼロ。

プロンプトを入れた瞬間、Claude が即座にコードを書き始めた。約1分後、330行の HTML ファイルが出てきて、ブラウザで開いたら動いた。

**良い点：** 最速。仕様通り。
**気になる点：** 技術スタック（HTML? React? Python?）をユーザーに聞かず Claude が全部決めた。結果的に正解だったが、「なぜこれにしたのか」の説明はない。テストもレビューもなし。

ユーザーの操作は「プロンプトを打つ」の1回だけ。判断ポイントはゼロ。

---

### superpowers：創造的。でも別のゲームができた。

`/brainstorm` と打つと、5つの創造的な方向性が提案された：

1. **星霜決闘** — 宇宙テーマ＋属性相性
2. **裏切りのジョーカー** — ブラフ＋心理戦
3. **時の砂時計** — 時間帯でルール変動
4. **墨闘師** — 和風水墨画テーマ
5. **進化する手札** — メタゲーム＋やり込み

「2」と答えたら、ブラフ心理戦のカードゲームが完成した。687行、CPU に性格パターンがあり、3D フリップアニメーション付き。

**良い点：** 圧倒的に面白いゲームが出てきた。5案から選べるのは楽しい。
**気になる点：** 元のプロンプト「数字が大きい方が勝ち」の単純なルールは消えて、まったく別のゲームになった。テスト・レビュー・保存のフェーズがない。また、`/brainstorm` と明示的に打たないとスキルが発動しない（何も打たなかった1回目は素の Claude Code と同じ動作だった）。

ユーザーの操作は3回。判断ポイントは実質1回（5案から選ぶ）。

---

### CEWK：4分で完成。全部「選ぶだけ」。

`/strategy` と打つと、こう聞かれた：

> **A: シングルHTML — ブラウザで即プレイ**
> **B: Python CLI — ターミナルで対戦**
> **C: React / Vite — モダンWebアプリ**
> おすすめ: A

「A」と答えたら、自動で /plan に進んだ。計画が出てきて：

> **やること：**
> 1. カードプールからランダム5枚ずつ配る
> 2. クリックでカード選択 → CPU が自動で出す
> 3. 数字を比較して勝敗判定
> 4. 履歴をリアルタイム表示
> 5. 5ラウンド後に最終結果 + リプレイ
> 6. ダークテーマ + アニメーション

「GO」と答えたら、自動で実装が始まった。コードが書かれ、そのまま /debug → /review が自動で回った：

- /debug：ロジック8項目を自動検証（カード配布、勝敗判定、スコア加算、リプレイ等）
- /review：セキュリティ・パフォーマンス・要件充足を9項目チェック → lgtm

「保存しますか？」と聞かれたので「ほぞん」（タイプミス）と答えたら、devlog に記録して完了。

**良い点：** 仕様通りのゲームが完成。品質検証済み。判断は全部「これでいい？」系。
**気になる点：** superpowers のような「もっと面白くする提案」はなかった。速さでは素の Claude Code に負ける。

ユーザーの操作は4回。判断ポイントは3回（A/B/C選択、計画承認、保存承認）。全て「はい」で通せる。

## 何が分かったか

### 1. 「文系フレンドリー」は判断の少なさではなかった

3環境とも、専門知識を要求するポイントはゼロだった。Claude Code は元からそこそこ文系フレンドリーだ。

差が出たのは **「仕様通りに完成するかどうか」** と **「完成物が検証されているかどうか」**。

### 2. superpowers の強みは「創造性の増幅」

5案のブレインストーミングから始まるフローは、「何を作るか決まっていない」段階で強い。ただし仕様が決まっているタスクでは、提案に引きずられて別のものができるリスクがある。

また、スキルが自動発動しない（明示的にコマンドを打つ必要がある）のは、知らない人には使えないということ。

### 3. CEWK の強みは「一貫性」

`/strategy` と打てば全工程が自動で回り、仕様通りに完成し、テスト・レビュー・記録まで済む。判断ポイントは3回あるが、全て「おすすめがある」「はい/いいえで答えられる」設計。

文系ユーザーにとって大事なのは「判断が少ない」ことではなく、**「判断を間違えても大丈夫」** と思えること。CEWK は全ての判断に「おすすめ」があり、不可逆な選択がない。

---

## まとめ

| ユースケース | おすすめ |
|-------------|---------|
| 「とにかく速く作って」 | 素の Claude Code |
| 「何を作るかから一緒に考えて」 | superpowers |
| 「仕様通りに、品質検証付きで作って」 | CEWK |

superpowers と CEWK は競合ではなく、**解いている問題が違う**。

superpowers は「開発者の創造性を拡張するツール」。
CEWK は「非開発者が安心して手順に乗るためのガードレール」。

文系ユーザーが「AI でアプリを作ってみたい」と思ったとき、最初に必要なのはガードレールのほうだ。創造性の拡張は、ガードレールの上に乗ってからでいい。
