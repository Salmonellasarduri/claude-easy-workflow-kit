# サブエージェント出力 schema（必須）

Agent tool / Explore / Plan / Codex / Gemini 等のサブエージェント呼出時のプロンプト末尾に付ける定型 schema。

---

## 必須の 4 項目

```
## 結論
（1-3 行で最重要発見）

## 根拠
（主要な参照パス・発見事項、5-15 行）

## 未確定点
（追加調査が必要な点、1-5 行）

## 参照先
（重要な file:line リスト、最大 10 件）
```

---

## プロンプト末尾に追加する定型文（コピペ推奨）

```
# 出力形式（必須）
最終出力は以下の 4 項目 schema のみで返してください:
- 結論（1-3 行）
- 根拠（主要な参照パス・発見事項、5-15 行）
- 未確定点（1-5 行）
- 参照先（file:line リスト、最大 10 件）

生出力の長文展開は禁止。400-600 token の構造化要約で返すこと。
```

---

## 適用範囲

- `Agent` tool（general-purpose / Explore / Plan / 各種 subagent）すべて
- `codex exec` 呼出（批評・レビュー系）
- `gemini -p` 呼出（調査・分析系）
- `commands/` 内のワークフローコマンドからのエージェント呼出も同じ schema を末尾に付ける

## 理由

サブエージェント結果の生出力垂れ流しは Claude Code セッショントークンの主要消費源。4 項目 schema を強制することで、`結論` と `参照先` だけを消費しつつ「詳細が必要なら `参照先` を直接 Read する」呼び出し方に自然に誘導できる。
