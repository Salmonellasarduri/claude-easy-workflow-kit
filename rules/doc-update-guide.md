# ドキュメント更新ガイド

`/save` 時にドキュメントを更新するためのチェックリストと規約。
ドキュメント更新は `/save` の VCS 操作とは **分離された前段ステップ** として実行する。

---

## 更新対象（ホワイトリスト）

| ファイル | 更新タイミング | 内容 |
|---------|--------------|------|
| `docs/devlog/YYYY-MM.md` | 毎セッション | セッション詳細を末尾に追記 |
| `docs/DEVLOG.md` | 毎セッション | 1行サマリーを先頭に追加（最新10件を保持） |
| `tasks/current.md` | タスク完了時 | 完了タスクに `[x]` を付ける |
| `docs/ROADMAP.md` | フェーズ完了時のみ | 状態を更新（構造は変更しない） |
| `README.md` | 機能追加時のみ | 関連セクションのみ更新 |

> パスは `workflow.yaml` の設定で変更可能。

---

## 更新ルール

1. **変更したセクションだけを更新する** — 触っていない部分は一切変えない
2. **既存エントリを編集・削除しない** — devlog は追記のみ
3. **Git 操作はしない** — ドキュメント更新と staging/commit は別ステップ
4. **月次ファイルが 1000行を超えたら** `YYYY-MM-part2.md` を新設して継続

---

## サブエージェントへの委譲テンプレート

大規模な変更の場合、ドキュメント更新を general-purpose サブエージェントに委譲できる:

```
以下の実装が完了しました。ドキュメントを更新してください。

## 実装内容
[何をしたか]

## 変更ファイル
- src/xxx.py
- src/yyy.py

## 日付・セッション
YYYY-MM-DD セッション N

## 更新指示
1. docs/devlog/YYYY-MM.md にセッション詳細を追記する
2. docs/DEVLOG.md の「最新セッション」に1行サマリーを先頭追加
3. tasks/current.md の完了タスクに [x] を付ける
4. 必要に応じて README.md を更新する

ホワイトリスト外のファイルは変更しないこと。Git 操作は行わないこと。
```

---

## 小規模な変更の場合

サブエージェントを使わず、`/save` の中で直接更新してもよい:

1. `tasks/current.md` の完了タスクに `[x]` を付ける
2. `docs/DEVLOG.md` に1行サマリーを追加する
3. 必要に応じて他のファイルを更新する
